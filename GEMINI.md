## 紀錄規範
- **變更日誌**：凡新增、修改、刪除元件，必須將 **Element ID** 及具體變更紀錄於 `output/mod_history.log`。
- **自檢報告 (QC Report)**：產出的自檢表需存於 `output/reports/`，命名格式為 `YYYYMMDD_QC_[類別]_[樓層].md`（如 `20260416_QC_Structural_B4F.md`）。
- **文件標準**：遵循 `knowledge/qc_report_standard.md` 定義的報告格式。

## 溝通與情緒規範
- **禁止個人情緒表達**：嚴禁使用感性、個人化或非關技術之情緒性用語（例如「懷疑人生」、「堅持到底」、「勝利」等）。
- **精簡專業**：所有回應必須維持絕對的專業性與客觀性，僅針對技術指令與執行結果進行表述。

## 操作邏輯 (三階段執行法)
1. **計畫 (Planning)**：
   - **超前思維**：預判後續邏輯所需之品類與型別轉換，一次性完成 MCP 工具擴充，避免重複編譯部署。

## 安全守則
- **禁止刪除軸網 (Grids) 與樓層 (Levels)**：嚴禁對 Axis 或 Level 執行刪除，僅能由使用者授權後進行讀取。
- **類型變更保障**：建立新類型 (Type) 前，必須確認樣板元件。
- **嚴格任務遵從 (Strict Task Adherence)**：
  - Agent 必須僅執行使用者該次輸入所指定的任務或指令（如 `/mcp-check`）。
  - **嚴禁過度執行**：即使心中有後續計畫，也絕對禁止在未獲當下明確授權的情境下，擅自跨入下一個邏輯步驟。
  - 任務完成後必須停下等待指令，不得「順便」執行任何寫入、移動或修改操作。

2. **執行 (Execution)**：
   - **強制斷點**：凡涉及 DLL 覆蓋或系統更新，必須先請使用者確認關閉 Revit 建立斷點，嚴禁在佔用狀態下執行部署。

## MCP 調用紀律
- **正式任務必須使用 MCP Tool**：Agent 執行自動化任務時，必須透過 MCP 協議調用工具（如 `mcp_REVIT_MCP_study_copy_document`），嚴禁直連 WebSocket (Port 8964)。
- **直連 WebSocket 僅限開發驗證**：以下情境可暫時使用 `node -e` 直連 WebSocket：
  1. 新功能部署後、MCP Server 尚未重啟時的冒煙測試。
  2. 開發者手動除錯 C# 端回傳格式。
- **底層等價性**：兩種調用方式最終進入同一個 C# 方法，功能完全一致；差異僅在傳輸層上游。
- **禁止汙染知識庫**：Skill 文件、日誌與報告中，不得將直連 WebSocket 記錄為標準操作流程。
